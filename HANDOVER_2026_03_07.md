# assemblykor - Handover (2026-03-07)

## Project Overview

**assemblykor** is an R package providing curated Korean National
Assembly datasets for teaching quantitative methods in political
science. Korean politics counterpart to `palmerpenguins`.

- **Version**: 0.1.0
- **Repository**: <https://github.com/kyusik-yang/assemblykor>
- **License**: MIT (code), data licenses vary (see README)
- **Local path**:
  `/Volumes/kyusik-ssd/kyusik-research/projects/assemblykor/`

## Technical Notes

- **CSS theming**: Custom dark theme (652 lines), Catppuccin Mocha
  inspired. Fonts: Inter (UI), JetBrains Mono (code). Uses
  [`htmltools::includeCSS()`](https://rstudio.github.io/htmltools/reference/include.html)
  in each tutorial Rmd, not YAML `css:` parameter.
- **Data pipeline**: `data-raw/prepare_data.R` (R) + `process_*.py`
  (Python). Raw source files (.gitignored):
  `speeches_all_committees.csv` (3.1GB), `speeches_full.parquet`
  (1.4GB), `roll_calls_raw.csv` (44MB).
- **Dependencies**: `arrow` required only for download functions. Core
  package depends only on base R + utils.
- **LazyData**: Compressed with xz. Package install includes all 7 .rda
  files.

## Known Data Characteristics

- `n_bills` in `legislators`: includes co-sponsorship. Median 702, max
  4,198. Can surprise students expecting only lead-proposed bills.
- `n_bills_lead`: lead proposals only. Median 56, max 696.
- 661 unique legislators across 947 rows (some serve multiple
  assemblies).
- `wealth` unit: thousands of KRW (divide by 1,000,000 for billions).
- `roll_calls` API only available for 22대.
